# Help Desk Ticket Workflow

This workflow describes the full lifecycle of a help desk ticket.

---

## 1. Ticket Creation
- User submits a ticket using the provided template  
- Ticket is assigned a unique Ticket ID  
- Technician reviews priority and category  

---

## 2. Initial Response
- Technician acknowledges ticket within SLA  
- Clarifies missing information  
- Documents initial notes  

---

## 3. Troubleshooting
- Follows standard troubleshooting procedures  
- Logs each step  
- Updates ticket status to *In Progress*  

---

## 4. Resolution
- Technician identifies and implements fix  
- Documents resolution clearly  
- Updates user  
- Changes status to *Resolved*  

---

## 5. Closure
- User confirms issue is fixed  
- Status changed to *Closed*  
- Ticket is archived
